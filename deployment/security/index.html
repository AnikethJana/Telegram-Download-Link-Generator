    üõ°Ô∏è StreamBot Security Framework¶

    StreamBot now includes a comprehensive security framework that addresses all major web application vulnerabilities. This guide covers deployment, configuration, and monitoring.

    üö® Security Status: IMPLEMENTED ‚úÖ¶

    StreamBot's security framework is fully implemented and ready for production use with:

    • ‚úÖ JWT Authentication with 256-bit entropy
    • ‚úÖ Advanced Rate Limiting with DDoS protection
    • ‚úÖ Input Validation and sanitization
    • ‚úÖ Security Headers (CSP, HSTS, XSS protection)
    • ‚úÖ Secure CORS configuration
    • ‚úÖ Request Size Limits and DoS protection
    • ‚úÖ Error Handling without information disclosure
    • ‚úÖ Log Security with sensitive data masking

    üîß Essential Security Configuration¶

    Critical Environment Variables¶

    # === JWT AUTHENTICATION (CRITICAL) ===
# Generate with: python3 -c "import secrets; print(secrets.token_urlsafe(32))"
JWT_SECRET=your_256_bit_jwt_secret_here_32_chars_minimum_required

# === ADMIN ACCESS CONTROL (CRITICAL) ===
ADMINS=123456789 987654321  # Space-separated Telegram user IDs

# === CORS SECURITY (CRITICAL) ===
# NEVER use "*" - specify exact domains
ALLOWED_ORIGINS=https://yourdomain.com,https://admin.yourdomain.com

# === HTTPS ENFORCEMENT (CRITICAL) ===
BASE_URL=https://yourdomain.com  # Must use HTTPS in production

# === DATABASE SECURITY ===
DATABASE_URL=mongodb://username:password@localhost:27017/streambot?authSource=admin

    Generate Secure Secrets¶

    # Generate all required secrets at once
echo "JWT_SECRET=$(python3 -c 'import secrets; print(secrets.token_urlsafe(32))')"
echo "LOGS_ACCESS_TOKEN=$(python3 -c 'import secrets; print(secrets.token_urlsafe(32))')"

# Alternative with OpenSSL
echo "JWT_SECRET=$(openssl rand -base64 32)"
echo "LOGS_ACCESS_TOKEN=$(openssl rand -base64 32)"

    Configuration Validation¶

    StreamBot automatically validates your security configuration on startup:

    # Check security status
python3 -m StreamBot

# Look for these log messages:
# ‚úÖ Security validation PASSED - No critical issues found
# ‚ö†Ô∏è Security validation: WARNING - 2 warnings found
# ‚ùå Security validation FAILED - Critical issues found!

    üèóÔ∏è Server Security Setup¶

    1. Operating System Hardening¶

    # Update system packages
sudo apt update && sudo apt upgrade -y

# Install security essentials
sudo apt install -y ufw fail2ban unattended-upgrades

# Configure automatic security updates
sudo dpkg-reconfigure unattended-upgrades

    2. Firewall Configuration¶

    # Configure UFW firewall
sudo ufw default deny incoming
sudo ufw default allow outgoing

# Allow SSH (change port if needed)
sudo ufw allow ssh
# sudo ufw allow 2222/tcp  # If using custom SSH port

# Allow HTTP/HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# Allow StreamBot port (if direct access)
sudo ufw allow 8080/tcp

# Enable firewall
sudo ufw enable
sudo ufw status verbose

    3. SSH Security Hardening¶

    # Edit SSH configuration
sudo nano /etc/ssh/sshd_config

# Apply these settings:
# PasswordAuthentication no
# PermitRootLogin no
# Protocol 2
# MaxAuthTries 3
# Port 2222  # Optional: Change default port

# Restart SSH service
sudo systemctl restart ssh

    4. Intrusion Detection (Fail2Ban)¶

    # Create StreamBot-specific fail2ban jail
sudo nano /etc/fail2ban/jail.local

    [DEFAULT]
# Ban for 10 minutes after 5 failures in 5 minutes
bantime = 600
findtime = 300
maxretry = 5

[streambot-auth]
enabled = true
port = 80,443,8080
filter = streambot-auth
logpath = /path/to/streambot/tgdlbot.log
maxretry = 3
bantime = 3600

[streambot-ddos]
enabled = true
port = 80,443,8080
filter = streambot-ddos
logpath = /path/to/streambot/tgdlbot.log
maxretry = 10
bantime = 7200

    Create filters:

    # Authentication filter
sudo nano /etc/fail2ban/filter.d/streambot-auth.conf

    [Definition]
failregex = ^.*Unauthorized.*from <HOST>.*$
            ^.*Failed authentication.*<HOST>.*$
            ^.*Invalid token.*<HOST>.*$
ignoreregex =

    # DDoS filter
sudo nano /etc/fail2ban/filter.d/streambot-ddos.conf

    [Definition]
failregex = ^.*Rate limit exceeded.*<HOST>.*$
            ^.*DDoS protection triggered.*<HOST>.*$
ignoreregex =

    # Restart fail2ban
sudo systemctl restart fail2ban
sudo systemctl enable fail2ban

    üåê Web Server Security (Nginx)¶

    Secure Nginx Configuration¶

    # /etc/nginx/sites-available/streambot
server {
    listen 80;
    server_name yourdomain.com;
    return 301 https://$server_name$request_uri;  # Force HTTPS
}

server {
    listen 443 ssl http2;
    server_name yourdomain.com;

    # SSL Configuration
    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;
    ssl_session_timeout 1d;
    ssl_session_cache shared:SSL:50m;
    ssl_session_tickets off;

    # Modern SSL configuration
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;

    # HSTS
    add_header Strict-Transport-Security "max-age=63072000" always;

    # Security headers (additional to StreamBot's built-in headers)
    add_header X-Robots-Tag "noindex, nofollow" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # Rate limiting
    limit_req_zone $binary_remote_addr zone=api:10m rate=10r/m;
    limit_req_zone $binary_remote_addr zone=download:10m rate=100r/m;

    # File upload limits
    client_max_body_size 100M;
    client_body_timeout 60s;
    client_header_timeout 60s;

    # Proxy to StreamBot
    location / {
        # Apply rate limiting
        limit_req zone=api burst=20 nodelay;

        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Timeouts
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # Optimized for downloads
    location /dl/ {
        limit_req zone=download burst=50 nodelay;

        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Longer timeouts for file downloads
        proxy_connect_timeout 120s;
        proxy_send_timeout 300s;
        proxy_read_timeout 300s;
        proxy_buffering off;
    }

    # Block access to sensitive files
    location ~ /\.(env|git|htaccess|htpasswd) {
        deny all;
        return 404;
    }
}

    Secure MongoDB Configuration¶

    // MongoDB security setup
use admin

// Create admin user
db.createUser({
  user: "admin",
  pwd: "your_secure_admin_password",
  roles: ["userAdminAnyDatabase", "dbAdminAnyDatabase", "readWriteAnyDatabase"]
})

// Create application user
use streambot
db.createUser({
  user: "streambot_app",
  pwd: "your_secure_app_password", 
  roles: ["readWrite"]
})

    # /etc/mongod.conf
security:
  authorization: enabled

net:
  port: 27017
  bindIp: 127.0.0.1  # Only localhost access

storage:
  wiredTiger:
    engineConfig:
      directoryForIndexes: true
    collectionConfig:
      blockCompressor: snappy
    indexConfig:
      prefixCompression: true

    # Restart MongoDB
sudo systemctl restart mongod
sudo systemctl enable mongod

    MongoDB Connection Security¶

    # Secure MongoDB connection string
DATABASE_URL=mongodb://streambot_app:your_secure_app_password@localhost:27017/streambot?authSource=streambot&ssl=false

    üîç Security Monitoring¶

    Real-time Security Dashboard¶

    StreamBot provides a built-in security monitoring endpoint:

    # Check security status
curl -H "Authorization: Bearer YOUR_JWT_TOKEN" \
     https://yourdomain.com/api/security

    Response includes: - Configuration validation results - Authentication statistics - Rate limiting status
    - Recent security events - DDoS protection status

    Log Monitoring Setup¶

    # Install log monitoring tools
sudo apt install logwatch logrotate

# Configure log rotation for StreamBot
sudo nano /etc/logrotate.d/streambot

    /path/to/streambot/tgdlbot.log {
    daily
    missingok
    rotate 30
    compress
    notifempty
    create 644 streambot streambot
    postrotate
        systemctl restart streambot || true
    endscript
}

    Security Event Alerting¶

    #!/bin/bash
# /usr/local/bin/streambot-security-monitor.sh

LOG_FILE="/path/to/streambot/tgdlbot.log"
ALERT_EMAIL="admin@yourdomain.com"

# Monitor for security events
tail -f "$LOG_FILE" | while read line; do
    if echo "$line" | grep -E "(CRITICAL|DDoS protection triggered|Failed authentication)"; then
        echo "SECURITY ALERT: $line" | mail -s "StreamBot Security Alert" "$ALERT_EMAIL"
    fi
done

    # Make executable and add to cron
chmod +x /usr/local/bin/streambot-security-monitor.sh

# Add to crontab
crontab -e
# @reboot /usr/local/bin/streambot-security-monitor.sh &

    Secure Dockerfile¶

    FROM python:3.11-slim

# Create non-root user
RUN groupadd -r streambot && useradd -r -g streambot streambot

# Install security updates
RUN apt-get update && apt-get upgrade -y && \
    apt-get install -y --no-install-recommends \
    ca-certificates && \
    rm -rf /var/lib/apt/lists/*

WORKDIR /app

# Copy requirements first (layer caching)
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy application
COPY . .

# Set secure file permissions
RUN chown -R streambot:streambot /app && \
    chmod -R 755 /app && \
    chmod 600 /app/.env

# Run as non-root user
USER streambot

# Expose port
EXPOSE 8080

# Health check
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
    CMD curl -f http://localhost:8080/ || exit 1

CMD ["python", "-m", "StreamBot"]

    üìä Performance vs Security Balance¶

    StreamBot uses selective security to maintain high performance:

    Download Route (/dl/) - Performance Optimized¶

    • ‚úÖ Global rate limiting (500/minute)
    • ‚úÖ Basic input validation (lightweight)
    • ‚úÖ Essential security headers
    • ‚úÖ Error handling without info disclosure
    • ‚ùå No per-client tracking (performance)
    • ‚ùå No heavy regex validation (performance)

    Admin Routes (/api/logs, /api/security) - Full Security¶

    • ‚úÖ JWT authentication with multi-factor verification
    • ‚úÖ Complete input validation and sanitization
    • ‚úÖ Per-IP rate limiting (10/minute)
    • ‚úÖ Full security headers (CSP, HSTS, etc.)
    • ‚úÖ Comprehensive logging and monitoring

    üö® Security Checklist¶

    Pre-Production Checklist¶

    • Secrets Generated: JWT_SECRET and LOGS_ACCESS_TOKEN (256-bit)
    • HTTPS Enabled: BASE_URL uses https://
    • CORS Configured: ALLOWED_ORIGINS set to specific domains (no wildcards)
    • Admin Access: ADMINS properly configured
    • Database Secured: MongoDB authentication enabled
    • Firewall Active: UFW/iptables configured
    • SSL Certificate: Valid SSL certificate installed
    • Fail2Ban Setup: Intrusion detection configured
    • Log Monitoring: Security event alerting configured
    • Security Validation: python -m StreamBot shows "PASSED"

    Post-Deployment Monitoring¶

    • Daily: Check /api/security endpoint for issues
    • Weekly: Review security logs for suspicious activity
    • Monthly: Update system packages and dependencies
    • Quarterly: Rotate JWT_SECRET and LOGS_ACCESS_TOKEN
    • As Needed: Monitor fail2ban logs and adjust rules

    üîß Troubleshooting Security Issues¶

    Common Security Configuration Problems¶

    1. JWT Secret Too Short¶

    # Error: JWT_SECRET too short (16 chars) - minimum 32 required
# Solution: Generate longer secret
JWT_SECRET=$(python3 -c "import secrets; print(secrets.token_urlsafe(32))")

    2. CORS Wildcard Detected¶

    # Error: CORS wildcard '*' detected - major security risk
# Solution: Specify exact origins
ALLOWED_ORIGINS=https://yourdomain.com,https://admin.yourdomain.com

    3. Non-HTTPS Base URL¶

    # Warning: BASE_URL should use HTTPS in production
# Solution: Enable HTTPS
BASE_URL=https://yourdomain.com

    Security Testing¶

    # Test rate limiting
for i in {1..60}; do curl -s https://yourdomain.com/api/info; done

# Test authentication
curl -H "Authorization: Bearer invalid_token" https://yourdomain.com/api/logs

# Test CORS
curl -H "Origin: https://malicious-site.com" https://yourdomain.com/api/info

# Check security headers
curl -I https://yourdomain.com/
